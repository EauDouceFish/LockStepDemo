【今日教学讨论中产生的关键认识】

- Component class vs struct：用 class（避免 Dictionary 装箱副本写不回）
- Entity ID 从 1（0 保留作 invalid 哨兵）
- Frame++ 在 foreach 之后（语义 = "已完成帧数"）
- Type 做 Dictionary key 不重（typeof 编译期常量），瓶颈在缓存不友好
- PascalCase 不是匈牙利规范（匈牙利是 iCount/m_iFrame，PascalCase 是 .NET 标准）
- LogTickSystem 用 Action<string> 而不是 ILogger（单职责场景接口最小化原则）
- Update 里调 Tick 是错的（频率漂移 + Time.deltaTime 不确定 → 状态分叉）
- Time.deltaTime 不能进逻辑层；累计器模式从 View 层吞 deltaTime
- 本地输入不直接喂 World（必须走"上行→服务器→下行权威"两条路）
- 服务器是邮局不是裁判（不跑游戏逻辑，只转发输入）
- FrameAuth 打包所有玩家本帧输入（一次广播原子语义）
