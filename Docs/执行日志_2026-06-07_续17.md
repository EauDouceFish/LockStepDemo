# 2026-06-07 续 17：展馆输入/连招说明 UI + 5110 倒地起身修复

用户要求按 TDD 粒度拆模块，并用 Unity MCP 截图自验。本轮按模块完成：

## 模块 A：连招说明数据模型

测试脚本：`Tests/Lockstep.Logic.Tests/Mugen/Command/CommandMoveCatalogTests.cs`

引擎预期：
- `.cmd [Command]` 的原始 `command = ...` 文本必须保存到 `MCommandDef.Motion`。
- `Statedef -1` 的 `command="..." -> ChangeState/SelfState value` 必须能生成可展示的招式说明。
- KFM `QCF_x` 显示为 `~D, DF, F, x -> state 1000`。

实现：
- `MCommandDef` 增加 `Motion`。
- `MCommandParser.Parse` 填入原始 motion。
- 新增 `MCommandMoveCatalog` / `MCommandMoveInfo`，把 command definition 与 transition entry 合并成 UI 可读目录。

验收：`CommandMoveCatalogTests` 2/2。

## 模块 B：输入面板

测试脚本：`Tests/Lockstep.Logic.Tests/Mugen/Command/CommandInputDisplayTests.cs`

引擎预期：
- `MInput.None` 显示为 `None`。
- 方向 + MUGEN 按键按固定顺序显示，例如 `Down|Right|X` -> `D + R + x`。
- 六键显示顺序为 `x + y + z + a + b + c`。

实现：
- 新增 `MInputDisplayFormatter`。
- `MugenMuseumDashboard` 左侧角色列表上方新增 Input 面板，显示 Current input、Active command、按键映射。
- GUI 手动按钮增加同帧防抖，避免一次 GUI 点击在同一帧重复消费。

验收：`CommandInputDisplayTests` 3/3。

## 模块 C：倒地自动起身

测试脚本：`Tests/Lockstep.Logic.Tests/Mugen/Battle/KnockdownRecoveryTests.cs`

引擎预期：
- KFM 从 5110 liedown 状态开始，应等待 `data.liedown.time` 后自动进入 5120 getup。
- 5120 动画结束后回到 state 0，`StateType=S` 且 `Ctrl=true`。
- 正常 `StartRound()` 不应开局落在 5110 或 LieDown。

实现：
- `MStateMachine.UpdateGetHitTimers` 增加 5110 硬编码起身计时：按 `MConstants.LiedownTime` 初始化/递减，到 0 后排队切 5120。

验收：`KnockdownRecoveryTests` 2/2。
