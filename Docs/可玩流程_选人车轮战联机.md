# 可玩流程：选人、车轮战、手机匹配联机

更新时间：2026-06-12。

## 当前结论

现在 `KihanDemo` 式的“外部两台手机搜寻对手、加入房间、Ready、开局、退出/取消”的 MVP 主干已经落到本项目里：

- 选人页 `MugenSelectScreen`：可切换联机模式，手机可点角色格子和按钮。
- 角色包加载 `MugenCharacterPackageLoader`：按主 DEF 解析 `sprite/anim/cmd/cns/st/stcommon`，支持大小写不一致和多 DEF 的稳定选择，比旧的 `kfm.*` 硬编码更接近“导入任意合法 DEF 角色”的入口要求。
- 匹配服 `MugenRelayServer` + `MugenMatchServerCore`：支持排队、两人房间、房间分配、Ready 后开局、输入转发、取消、离房、超时关闭。
- 战斗页 `MugenTeamBattleView`：复用匹配阶段的 KCP 连接，等待 `StartMatch` 后进入 delay lockstep。
- 手机输入 `MugenMobileInput`：按 KiHan 的虚拟摇杆思路做了 uGUI 摇杆，按钮扩到 A/B/C/X/Y/Z/S，并在暂停/隐藏时清空输入。
- 锁步 `MugenLockstepSession`：加入 backpressure，远端长时间缺输入时不会继续覆盖未模拟帧。
- 资源一致性：客户端发送协议/common 级兼容 hash，服务端只匹配 `ClientVersion` 和 `ContentHash` 一致的玩家；不再因为本地多一个无关角色而无法匹配。

## 手机端流程

1. 打开 `Assets/Scenes/MugenTeamSelect.unity`。
2. 在选人页切到联机模式：
   - 键盘按 `3`，或点左下角“模式”按钮循环到 `NetKcp`。
3. 点选自己队伍的 3 个角色。
4. 点“开始”进入搜寻。
5. 第二台手机执行同样流程。匹配成功后，两端会自动进入 `TeamBattle`。
6. 战斗页会先显示等待对手，双方都发出 `RoomReady` 后服务端下发 `StartMatch`，再开始模拟。
7. 退出战斗会发 `LeaveRoom`；选人页取消会发 `CancelMatch`；房间关闭后可按 `Esc` 回到选人页。

键位：

- 摇杆：方向。
- A/B/C/X/Y/Z/S：对应 MUGEN/Ikemen 的七个按钮位。
- 桌面调试仍可用方向键 + `Z/X/C`。

## 本地调试

1. 打开 `Assets/Scenes/MugenRelayServer.unity`，Play，默认 UDP `7777`。
2. 另开两个客户端实例进入 `MugenTeamSelect.unity`。
3. 两端的 `MugenSelectScreen.ServerHost` 指向服务器 IP：
   - 本机同机调试：`127.0.0.1`
   - 真机局域网：电脑局域网 IP
   - VPS：`47.84.193.58`
4. 两端选择 3 人队并点“开始”。

## 新加坡 VPS 部署方案

目标机器：`dev@47.84.193.58`。目前本机 SSH 免密/密码非交互登录失败，错误是 `Permission denied (publickey,password)`，所以还不能由 Codex 直接上传部署。需要先把本机公钥加入服务器 `~/.ssh/authorized_keys`，或你手动登录后执行部署命令。

推荐目录：

```bash
/home/dev/mugen-server/
  MugenServer.x86_64
  MugenServer_Data/
  logs/
```

Unity 构建建议：

1. Build Settings 只放 `Assets/Scenes/MugenRelayServer.unity`。
2. 平台选 Linux x86_64。
3. 勾 Dedicated Server/Headless，或启动参数使用 `-batchmode -nographics`。
4. 上传构建产物到 `/home/dev/mugen-server`。

服务器需要放行：

```bash
sudo ufw allow 7777/udp
```

systemd 示例：

```ini
[Unit]
Description=Mugen KCP Match Server
After=network.target

[Service]
User=dev
WorkingDirectory=/home/dev/mugen-server
ExecStart=/home/dev/mugen-server/MugenServer.x86_64 -batchmode -nographics -logFile /home/dev/mugen-server/logs/server.log
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启用：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now mugen-server.service
sudo systemctl status mugen-server.service
```

## 当前限制

- 现在是 2 人房间、1v1 车轮战，不是 10 人同局。10 人派对规模目前指“几间房同时玩”，不是同一房 10 人乱斗。
- 服务端是内存房间，重启会丢队列/房间；demo 级别可接受。
- 没有账号、鉴权、断线重连和房间列表。
- 角色包发现已经不再硬编码 `kfm.*`，但“任意 MUGEN 角色完整战斗等价”仍取决于控制器、触发器、SFF/AIR 长尾格式、声音和 Ikemen trace 对账。
- 资源 hash 是 MVP 版：只覆盖协议/common 基线，避免两端本地无关角色差异阻断匹配；如果对端选择了本机缺失的角色，当前仍会在进入战斗时加载失败，后续需要用户可见的导入兼容报告。
- 没做 UDP NAT 诊断 UI；外网真机失败时优先检查云安全组、防火墙和运营商网络。
