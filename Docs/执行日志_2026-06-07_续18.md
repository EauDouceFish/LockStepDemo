# 2026-06-07 续 18：展馆 one-shot 招式预览 + 真实缺口清单压缩

## 模块 A：状态号解析回归

问题：
- 合法 CNS 头 `[Statedef 5150, 0]` 之前被解析成 state `0`。
- 结果是 5150/5188 倒地/死亡状态控制器混入站立 state0，展馆第一帧会把部分角色切到倒地/死亡动画。

修复：
- `MugenCnsParser` 状态头解析改为取 `Statedef` 后第一个整数。
- 新增 parser 回归测试，保证 `[Statedef 5150, 0]` 不再覆盖 state0。

## 模块 B：展馆招式按钮 one-shot 预览

问题：
- 底部招式按钮原先在一次 GUI 点击里同步喂完整输入序列和尾帧。
- command active/buffer 会残留；当角色回到 state0 后，`Statedef -1` 可能再次消费同一个 active command，表现为同一 state 一直重复。

修复：
- 新增 `MCommandList.ResetRuntime()`：清空 command active、输入历史和 runtime 进度，但保留命令定义。
- 新增 `MMovePreviewSession`：逐帧播放合成输入脚本；进入目标 state 后清 command runtime，并结束输入脚本，避免重复触发。
- `MugenMuseumDashboard` 改为在 `Update()` 的 60Hz tick 中消费 preview session，不再在按钮点击瞬间同步推进整段技能。

测试：
- `CommandList_ResetRuntimeClearsActiveCommandAndInputHistory`
- `MovePreviewSession_PlaysButtonMoveOnceThenReturnsToDefault`
