> 最后更新：2026-06-02

## 0. 工作法（贯穿全程，所有 agent 必守）

### 0.0 战斗模式（范围锁定）
**v1 全程只做 MUGEN-2D 模式**（X 横向 + Y 高度，MUGEN 原生坐标，纯 2D 平面对拳）。火影纵深 3D 是 v2 独立支路，21 天内**不碰**。模式专属逻辑隔离在 4 个 seam（PhysicsSystem / CollisionSystem / 输入映射 / View 投影），命名带 `Mode2D`，共享层不掺模式判断。详见 `架构设计.md` §3b。

### 0.1 测试驱动（TDD）
每个任务单元：**先写会失败的测试 → 实现 → 全绿才算完**。没有测试的代码不予合并。

### 0.2 三类测试
1. **单元测试**：单个 controller / trigger / 解析器的行为。
2. **差分测试（核心方法论）**：以 **Ikemen GO 为 Oracle**。在 Ikemen 跑同一角色 + 同一输入序列，录下逐帧结果，编码成本引擎的期望值。对齐粒度：
   - **离散量**（当前状态号、动画帧号、命中/落空/格挡、KO、Ctrl）→ **必须完全一致**。
   - **连续量**（位置、速度）→ **容差内一致**（定点 vs 浮点，给 epsilon）。
   - 这把"调试"变成"对答案"，**单人 + agent 可自动收敛**，不靠肉眼判断手感。
3. **确定性黄金哈希**：固定输入序列跑 N 帧，断言 `World.ComputeHash()` == 预录黄金值。任何破坏确定性/改变行为的改动→红。**这是"收敛"的客观标尺。**

### 0.3 测试工程（脱 Unity）
逻辑层是纯 C#（`noEngineReferences`）。建独立 .NET 测试工程 `Tests/Lockstep.Logic.Tests`，**引用 Logic 源码**，`dotnet test` 秒级跑、不开 Unity。agent 验收只跑它。

### 0.4 多 agent 协议
- 一个 agent 认领一个 `T*`，在 **git worktree 隔离**中工作（已 git init）。
- 完成定义（DoD）：① 该任务测试全绿 ② 整套 `dotnet test` 全绿（不回归）③ 黄金哈希测试通过 ④ 符合腾讯 C# 规范 ⑤ 写 commit + ledger。
- 任务间依赖见各阶段"依赖"。无依赖的任务可并行多开。

### 0.5 提交规范（commit = ledger）
- 每个工作单元一个 commit。格式：
  ```
  <type>(<模块>): <一句话做了啥>  [T<编号>]

  - 关键改动点
  - 验收：跑了哪些测试、结果
  Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
  ```
  `type` ∈ feat/fix/refactor/test/chore/docs。
- 同时在 `Docs/执行日志.md` 追加一条（人类可读摘要，带绝对日期）。

## Phase 2 · 表达式 VM + 数据驱动状态机（D5–7，里程碑 M-引擎）

**目标**：删掉硬编码 `PlayerStates`，换成**数据驱动** `StateMachineSystem`；从 `common1.cns` 把 站/走/蹲/跳 作为**数据**跑起来。

| T | 任务 | 验收 |
|---|---|---|
| T2.1 | `IExpressionVM` 扩到 §10 的 ~40 trigger + 运算符（编译期 parse → 运行期定点求值） | 单测覆盖每个 trigger；差分对齐 Ikemen 求值 |
| T2.2 | `ICmdImporter`：`.cmd` → `CommandData` + State -1 规则 | 单测：解析 Terrarian.cmd 指令数/motion 对 |
| T2.3 | `ICnsImporter` v1：`.cns` → `StateDef + Controller`（子集 controller），常量表 | 单测：解析 common1 的 Statedef 0/20/40 结构对 |
| T2.4 | 通用 `StateMachineSystem`（跑 -1 + 当前状态 controllers，VM 求值） | 删除硬编码 State 类后行为等价 |
| T2.5 | ~20 controller 实现（§10），每个独立可测 | 每个 controller 单测 + 差分对齐 Ikemen |
| T2.6 | `PhysicsSystem`（按 Physics 类型）+ `AnimSystem` | 走路位移/跳跃抛物线 差分对齐 |

**依赖**：T2.4 依赖 T2.1；T2.5 可在 T2.4 后**多 agent 并行**（一人几个 controller）；T2.3 依赖 schema。
**这是最大的一块，也是最适合多开 agent 的一块。**

## Stretch · S13 回滚（超出 21 天）

框架已预留（§架构设计 9）。实现 = 新增 `RollbackPredictor`：缺输入预测推帧 → 收到真输入与预测不符 → `World.Restore` → 重算。验收：日志输出回滚次数/深度/误预测率，弱网不冻结。参考 Ikemen `rollback.go`。

## 关键风险与对策

| 风险 | 对策 |
|---|---|
| 浮点→定点转换导致行为偏离 MUGEN | 差分测试容差对齐；离散量必须全等 |
| controller 长尾无穷 | 严格按"KFM 实际用到"优先，子集够跑就停 |
| 多 agent 改动互相冲突 | worktree 隔离 + 黄金哈希测试兜底回归 |
| CNS 复杂角色导入失败 | 先 KFM/规整角色；复杂角色人工清洗或暂缓 |
| 21 天做不完回滚 | 回滚明确列 Stretch，不挤占主线 |
