# 2026-06-08 展馆状态预览修复

## 结论

展馆没有新写一套底层状态机；`MugenLiveView` 与 `MugenMuseumDashboard` 都走 `MBattleEngine/MStateMachine`。本次卡动作问题来自展馆外层输入/preview 驱动：

- 顶部轻拳/轻脚按钮此前仍在 IMGUI 回调中同步 Tick，多帧推进路径和键盘路径不一致。
- 招式 preview 进入目标 state 后如果 CNS 没自然 `ChangeState` 回默认态，UI 没有清晰 timeout 诊断。
- 换角色/重置时旧 preview/手动输入队列需要清理。

## 修改

- `MMovePreviewSession`
  - 只负责逐帧合成输入、检测进入目标 state、等待自然恢复。
  - 新增 `TimedOutSearching`、`TimedOutInMoveState`、`TimeoutStateNo/AnimNo/Time`、`StatusText`。
  - 不再在 Logic 层强制 `QueueTransition(0)`，避免污染 Ikemen/MUGEN 战斗语义。

- `MugenMuseumDashboard`
  - 新增 `_manualInputs`，所有按钮输入统一由 `Update()` 定步消费。
  - 顶部“轻拳 x / 轻脚 a”优先按导入 CMD 命令和 Statedef -1 目标 state 启动 preview。
  - 展馆层收到 preview timeout 后只做 presentation recovery，并在面板显示 timeout 状态。
  - LoadSession 清理旧 preview、手动输入队列和 preview 状态。

- `CommandMoveTests`
  - 新增不会自然收招的合成状态测试：preview 必须报告 timeout，不能伪造回 state0。
  - 新增 `target -> recovery -> state0` 测试：preview 离开目标 state 后必须继续等待自然恢复。
